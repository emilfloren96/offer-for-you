# Project Structure

## Root files

| File | Description |
|------|-------------|
| `package.json` | Frontend dependencies and scripts (Vite, React, Three.js, Tailwind) |
| `vite.config.ts` | Vite build configuration and base URL settings |
| `tsconfig.json` | TypeScript root config |
| `tsconfig.app.json` | TypeScript config for the frontend app source |
| `tsconfig.node.json` | TypeScript config for Node/Vite tooling |
| `eslint.config.js` | ESLint rules for the project |
| `.gitignore` | Files and folders excluded from git |

---

## `src/` — Frontend application (React + TypeScript)

Main source code for the user-facing app.

| File | Description |
|------|-------------|
| `main.tsx` | App entry point — mounts the React app into the DOM |
| `App.tsx` | Root component — handles routing between the configurator and company dashboard, manages global state (shape, materials, offers), and renders the landing page |
| `index.css` | Global CSS variables (colours, border radius) and base styles |

### `src/components/` — UI components

| File | Description |
|------|-------------|
| `Header.tsx` | Top navigation bar with logo, "Skapa offert" and "Företag" buttons |
| `ModelViewer.tsx` | Interactive 3D house viewer (Three.js / React Three Fiber) — lets the user click on building parts like roof, walls, windows |
| `MaterialPanel.tsx` | Side panel shown after clicking a building part — displays available materials with prices so the user can make a selection |
| `CostSummary.tsx` | Sticky bottom bar showing the running total cost of all selected materials |
| `OfferPanel.tsx` | Offer request form — collects customer contact info and submits the offer to the backend API |
| `CompanyAuth.tsx` | Login form for companies/craftsmen to access the company dashboard |
| `CompanyDashboard.tsx` | Dashboard for logged-in companies — shows incoming offer requests from homeowners |
| `QuickPost.tsx` | Modal form for homeowners to quickly describe a job in plain text (alternative to the 3D configurator) |

### `src/data/` — Static data

| File | Description |
|------|-------------|
| `materials.ts` | All available materials per building category (roof, walls, windows, etc.) with prices, units, and colours — also exports helper types and the `deriveQuantity` utility |

---

## `server/` — Backend (Node.js + Express + SQLite)

REST API that stores and retrieves offer requests.

| File | Description |
|------|-------------|
| `index.js` | Express server — defines API routes (`POST /api/offers`, `GET /api/offers`, company auth endpoints) and starts the HTTP server |
| `db.js` | SQLite database setup using `better-sqlite3` — creates tables and exports query helpers |
| `package.json` | Server-only dependencies (Express, better-sqlite3, cors) |
| `products.db` | SQLite database file — stores submitted offers and company accounts |

---

## `public/` — Static assets served as-is

| Path | Description |
|------|-------------|
| `public/models/` | 3D model files (`.glb`) for each house shape and floor count (rectangular, L-shaped, U-shaped, T-shaped — one and two floors) |
| `public/vite.svg` | Default Vite favicon |

---

## `dist/` — Production build output

Generated by `npm run build`. Not committed to git. Contains the compiled and bundled frontend ready for deployment.

---

## `node_modules/` — Installed dependencies

Auto-generated by `npm install`. Not committed to git.

- Root `node_modules/` — frontend packages
- `server/node_modules/` — backend packages
